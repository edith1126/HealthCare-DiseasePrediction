def age_bucket(age):
    if (age>=0)  & (age < 18):
        return 'child'
    elif (age >= 18) & (age <= 30):
        return 'young_adult'
    elif (age > 30) &  (age <= 65):
        return 'adult'
    elif age >= 65:
        return 'older'

diaa['age_group'] = diaa['age'].apply(age_bucket)

def married_status(status):
    if status == 'MARRIED' or status == 'MARRIED IN ROUND':
        return 'MARRIED'
    elif status == 'DIVORCED' or status == 'DIVORCED IN ROUND' or status == 'SEPARATED' or status == 'SEPARATED IN ROUND':
        return 'DIVORCED'
    elif status == 'WIDOWED' or status == 'WIDOWED IN ROUND':
        return 'WIDOWED'
    else:
        return 'NEVER MARRIED'

diaa['married_status'] = diaa['married'].apply(married_status)

Age and marrital status is somehow correlated. Patients who have had widowed are the most likely to have diabetes because they tend to be older, which put them at higher risk of diabetes. Patients who have never had married tend to be younger and less likely to have diabetes.

from sklearn.metrics import confusion_matrix
threshold = 0.5
predicted = (clf.predict_proba(X_test)[:,1] >= threshold).astype('int')

tn, fp, fn, tp = confusion_matrix( y_test, predicted).ravel()

recall = tp/(fp + tp)
precision = tp/ (tp + fn)
print('Precision:{}'.format(recall))
print('Recall:{}'.format(precision))

Precision:0.8611111111111112
Recall:0.4832134292565947

Precision: Out of all the ones predicted positive, 86% of them actually have diabetes. Recall: Out of all the ones actually positive, we predicted 48% of them have diabetes.

Overall, the area under curve is 0.82 using logistic regression. It performs better than random forest and also gives us transparent into what medications and how much it affects the prediction.

The modeltells us that whether the patients take GLIPIZIDE, GLYBURIDE or METFORMI is highly indicative of diabetes patients. FUROSEMIDE and ENSURE are also mildly indicative of diabetes. Besides medications, variables including age, sex and marrital status, are included in the model as well.

By setting the threshold at 0.5, the model has high precision of 0.86 and acceptable recall of 0.848. The model predicts the patients with diabetes only when it is certain that this might be the case. Good thing we are not scarying patients away with the false positive. We are able to catch about 48% of the patients, about half of the patients, who actually have diabetes. If we do not want to miss out on potentially diabetes patients, we can try lowering the threshold.
